## What This Workspace Produces

Each project bid lives in **its own subfolder** with:
- `BID.md` — the actual bid text (500–700 chars, winning tone)
- `PLANNING.md` — project breakdown + milestone plan
- `DEMO.md` or demo code/files — proof of capability linked in the bid
- `README.md` — shareable via GitHub/Pages

## Key Working Patterns

- **Every bid in its own folder** — no loose files at root level
- **Each folder is its own git repo** (or at minimum its own commit context)
- **Always create a demo or strategic document** to link in the bid
- **Bid text = 500–700 chars** — tight, winning, specific to the client's pain
- **Read `.planning/STATE.md` every session** to avoid context drift
- **Use workflows** for repetitive actions (find-projects, new-bid)
